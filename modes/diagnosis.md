# Mode: diagnosis — Single-Pass Diagnostic Evaluation

**Trigger:** User invokes `/claudemed diagnosis` explicitly.
**Goal:** Fast single-pass evaluation without full debate rounds. Useful for straightforward cases or when the user wants a quick read.

## 1. Safety Triage

Check for red flags (see `_shared.md`). Issue URGENT warning if present.

## 2. Case Intake

Extract from provided text:
- Chief complaint and duration
- Associated symptoms
- Any provided vitals, labs, or exam findings
- Relevant history from `_profile.md`

## 3. Clinical Reasoning

Use the most likely specialist domain for this complaint. Apply:
- **Pre-test probability** based on patient demographics + history
- **Pattern recognition** — does this fit a classic clinical syndrome?
- **Bayesian updating** — how does each piece of data shift the probability?

## 4. Report

Generate a markdown report in `reports/` with filename `{###}-{slug}-{date}.md`.

Use this exact format:

```markdown
# [Chief Complaint] — Diagnostic Evaluation
**Date:** [YYYY-MM-DD]
**Case #:** [###]
**Mode:** Single-pass (no debate)

---

## Presenting Data
[Bulleted summary of symptoms, timeline, relevant history]

---

## Clinical Assessment

Confidence: [X.X]/5

**Supporting evidence:**
- [Finding 1]
- [Finding 2]

**Against:**
- [What doesn't fit]
```
